## 零、最小化全功能验收路径（先读这章）

**目的**：用**最少的容器**、**最少的手机↔电脑往返**、**一遍挂载测多项**，把全部功能跑一遍。按本章顺序走，比逐章跳着测省一半时间。

### 0.1 一次性准备（照做即省事）

**电脑侧桌面 VC 造 4 个容器**（全用 AES / SHA-512，除非该项要测算法）：

| 编号 | 文件系统 | 大小 | 特殊 | 塞入文件 | 用途 |
|---|---|---|---|---|---|
| 容器① | FAT | 30MB | 带 **keyfile**（选 2 个文件当钥匙）+ **隐藏卷** | 外层塞 `outer.txt`，隐藏卷塞 `hidden.txt` | A1+C1 一容器双测 |
| 容器② | exFAT | 50MB | 塞一个 **20MB 大文件** `big.bin` | `exfat.txt` + `big.bin` | D2 + 簇泄漏回归 |
| 容器③ | NTFS | 50MB | 无 | `ntfs.txt` | D3 + NTFS 实验 |
| 容器④ | FAT | 30MB | **废容器**（可损毁，供救砖/改密练手） | `victim.txt` | A2 + B1 破坏性测试 |

**手机侧**：装本轮 APK（HEAD `53a2157` 的 debug）。容器①②③各在电脑上**完整复制一份备份**（写入类测试红线）。

**这 4 个容器 + 1 个 app 自建容器（B2 建）= 覆盖全部功能。** 无 root 设备跳过 0.6 节即可。

### 0.2 无需容器的功能（先测，最快，不碰加密链路）

一口气测完，全在 app 内，不用电脑：

1. **X1 Material You 动态色**：换手机壁纸 → 重开 app → 主题色跟随壁纸变（Android 12+）。旧系统走靛蓝兜底，不崩即可。
2. **X2 PIN 门禁**：设置 → PIN 锁 → 设置 PIN（如 `1234`）→ 杀进程重开 → 应弹 PIN 门禁 → 输对进入、输错红字不清空。
3. **X4 生物识别**：接着上一步（已设 PIN）→ 设置里开「生物识别解锁」→ 杀进程重开 → 应自动弹指纹 → 验指纹进入；取消指纹 → 可退回手输 PIN。
4. **X3 Panic PIN**：设置 → Panic PIN → 设一个**不同于解锁 PIN** 的值（如 `9999`）→ 记一个收藏容器 → 杀进程重开 → 门禁输 `9999` → 预期：app 直接退出，重开后 PIN 门禁**已被清空**（擦除生效）、收藏**已清空**。⚠️测前别放重要收藏。
5. **16 国语言**：系统语言依次切几种（至少验中/英/日/阿拉伯 RTL）→ 重开 app → 界面文案随之切换；**阿拉伯语重点看布局是否右对齐（RTL）**、不串位。

> 测完把 PIN / Panic PIN 都清掉，避免干扰后续容器测试。

### 0.3 一个容器串多测：容器①（A1 keyfile + C1 隐藏卷）

一次准备，测两项（详细判据见第二章 A1 / C1）：

1. 选容器① → 输**外层**密码 + 选**那 2 个 keyfile** → 解锁 → 见 `outer.txt` ✓（**A1 keyfile 解锁**通过）
2. 故意**少选 1 个 keyfile** → 应打不开 ✓（keyfile 全集匹配）
3. 上锁 → 重选容器① → 输**隐藏卷**密码（无需 keyfile，除非隐藏卷也设了）→ 解锁 → 见 `hidden.txt` ✓（**C1 隐藏卷解锁**通过）

### 0.4 一次挂载测多项：容器②（D2 exFAT + X5 媒体 + E 自动锁定 + 收藏）

解锁容器②后**别急着上锁**，一口气测：

1. 解锁 → 文件系统显示 exFAT，见 `exfat.txt` + `big.bin` ✓
2. **X5 加密相册/媒体**：往容器②先用 app 导入一张图片 + 一段短视频 → 点图片走**全屏画廊**（双指缩放）、点视频走 **ExoPlayer 播放** → 用完退出 ✓
3. **D2 簇泄漏回归**：删 `big.bin`（20MB 连续分配大文件）→ 记住"删了大文件" ✓
4. **导入/删除/覆写**：导入 `phone_exfat.txt`、删 `exfat.txt` ✓
5. **多容器收藏**：把容器②加入收藏 → 应出现在主页收藏列表 ✓
6. **E2 切后台锁**：设置确认「切后台锁」开 → 按 Home → 切回 → 应已上锁 ✓
7. 重新解锁 → **E1 超时锁**：超时档选 1 分钟 → 不动手机等 1 分钟 → 应自动上锁 ✓
8. 上锁 → 容器②传回电脑 → 桌面 VC 挂载：`big.bin`/`exfat.txt` 已删、`phone_exfat.txt` 在 → `chkdsk` 干净、**空闲空间正确恢复**（否则 X4 BUG-1 簇泄漏）✓

### 0.5 破坏性测试用废容器：容器④（A2 救砖 + B1 改密）+ B2 创建

**先测 B2（app 建容器，无破坏风险）**：
1. 主页 → 创建新容器 → AES/SHA-512/10MB → 熵页涂抹到满 → 创建 → 塞几个文件
2. 传回电脑 → 桌面 VC 挂载 + `chkdsk` 干净 ✓（**D4 回验**一并完成）

**再拿容器④做破坏性测试**（详见 A2 / B1）：
1. **A2 备份**：卷头工具 → 导出卷头备份（128KB 文件）✓
2. **B1 改密**：容器④输旧密码 → 改密码入口填新密码 → 存救援文件 → 执行 → 桌面 VC 新密码能开、旧密码不能开、`victim.txt` 完好 ✓
3. **A2 救砖**：电脑上 hex 改乱容器④开头 → 桌面 VC 确认打不开 → 手机救砖 → 桌面 VC 又能开 ✓

### 0.6 root 增强验收（仅 root 设备，无 root 跳过）

需 Magisk/su 真 root（Shizuku 不够，读不了 `/dev/block`）。详见路线图阶段 F：

1. **F1 root 门面**：root 设备上，解锁区/主页应出现 root 专属入口（块设备解锁/暴露）；非 root 设备这些入口**不出现** ✓
2. **F2 块设备解锁**：选一个 `/dev/block/...` 分区（**先拿可丢弃的外置 SD/U 盘**）→ 输密码 → 只读解锁 → 能浏览 ✓；只读过再测可写 ✓
3. **F3 系统暴露**：已挂载 + root → 「暴露到系统」→ 填目标目录 → 开始 → 别的文件管理器能看到明文 → 用完「清理」✓（⚠️高风险，知情同意）
4. **F4 保活前台服务**：解锁后应出现常驻通知；从最近任务划掉 app → 应触发紧急上锁 ✓

### 0.7 最小化路径总结

| 走完这几步 | 覆盖功能 |
|---|---|
| 0.2 无容器 | X1/X2/X3/X4 + 16 国语言 |
| 0.3 容器① | A1 keyfile + C1 隐藏卷 |
| 0.4 容器② | D2 exFAT + X5 媒体 + E1/E2 自动锁 + 收藏 |
| 0.5 容器④+B2 | A2 救砖 + B1 改密 + B2 创建 + D4 回验 |
| 0.6 root（可选） | F1/F2/F3/F4 |

**最少 5 个容器（①②③④ + app 自建）跑完除 D1/D3 外全部功能。** D1（FAT16/32）和 D3（NTFS）因涉及不同文件系统 chkdsk，仍需单独按第三章走。NTFS 记得先开设置里的「NTFS 实验」开关（第八章）。

## 一、环境准备

### 1.1 你需要
- 一加手机（安卓 16），装本轮 APK：`Sealchest-exFAT-create.apk`（在桌面）
- 电脑上的桌面版 VeraCrypt（1.26.x，与我们上游 pin 的 1.26.29 同代）
- Windows 机器跑 `chkdsk X:`（X = 桌面 VC 挂载盘符）
- 一个能在手机/电脑间传文件的方式（数据线 / 网盘 / adb push）
- **测试用密码，别用你真实密码**；测试容器建小的（10-50MB）

### 1.2 手机上给容器授权（重要）
- 选容器时，系统会问读权限——必须给
- 想测**写入类功能**（改密码 / 创建隐藏卷 / 救砖 / 删文件 / 覆写），选容器时还得让它拿到**写权限**：
  这些入口只有在"选中的容器 URI 拿到了持久写权限"时才显示。拿不到写权限，这些按钮不出现（不是 bug）。
- 若某入口没出现，先确认：选的是可写位置的容器（如手机内部存储 / SD 卡），不是只读来源（如某些云盘挂载）

### 1.3 算法 / PRF 编号对照（排查时用）
创建容器时选的算法/PRF，对应 VeraCrypt 编号：

| 算法 | 编号 | | PRF | 编号 |
|---|---|---|---|---|
| AES | 1 | | SHA-512 | 1 |
| Serpent | 2 | | Whirlpool | 2 |
| Twofish | 3 | | SHA-256 | 3 |
| Camellia | 4 | | BLAKE2s | 4 |
| Kuznyechik | 5 | | Streebog | 5 |

解锁时可选"自动"PRF（挨个试）；创建时必须指定确定 PRF。

### 1.5 本轮新增功能（v0.7，须一并验收）
X1-X5 体验层 + F1-F4 root 增强 + 16 国语言，验收步骤见新增的**第零章「最小化全功能验收路径」**。

### 1.4 文件系统编号对照（第三章 D 节用）
桌面 VC 创建容器时选文件系统，对应 app 内部识别：

| 文件系统 | 桌面 VC 创建时选项 | app 内部识别 | 单文件上限 | 备注 |
|---|---|---|---|---|
| FAT16 | FAT（卷 ≤ 2GB 默认） | FAT16 | 2GB | 小卷，老兼容 |
| FAT32 | FAT（卷 > 2GB 默认） | FAT32 | 4GB | 大卷通用 |
| exFAT | exFAT | exFAT | 无上限 | 大文件首选 |
| NTFS | NTFS | NTFS | 无上限 | 桌面 VC 常用 |

## 二、待验功能清单（建议按此顺序测）

优先级：先测**只读/解锁类**（不改容器，安全），再测**写入类**（会改容器，先拿废容器练手）。

### 【A1】keyfile 解锁　★只读类
**做什么**：用密码 + 一个或多个 keyfile 解锁容器。

**准备**：桌面 VC 造一个带 keyfile 的容器（创建时勾 Keyfiles，选定几个文件当钥匙）。记住密码 + 那几个 keyfile。

**手机步骤**：
1. 选中该容器 → 输密码
2. 点"密钥文件（可选）"→ 选中**同一组** keyfile（顺序无所谓）
3. 解锁

**预期**：能开，能浏览里面的文件。

**关键对照**：
- 同一组 keyfile 在手机和桌面结果必须一致（都能开或都不能开）
- 少选/多选一个 keyfile → 应该打不开（keyfile 是全集匹配）
- 纯 keyfile 无密码：VC 允许空密码 + keyfile，我们也放开了空密码解锁

**失败排查**：若桌面能开、手机不能开 → 可能是 keyfile 混入算法（CRC32 池化）有偏差，记下用了几个 keyfile、文件多大，告诉我。

---

### 【C1】隐藏卷解锁　★只读类
**做什么**：同一个容器，输外层密码开外层卷，输隐藏卷密码开隐藏卷。

**准备**：桌面 VC 造一个带隐藏卷的容器（Create → Hidden Volume）。外层放几个诱饵文件，隐藏卷放几个不同的文件。记两个密码。

**手机步骤**：
1. 选中容器 → 输**外层**密码 → 解锁 → 应看到诱饵文件
2. 上锁 → 重新选中 → 输**隐藏卷**密码 → 解锁 → 应看到隐藏卷的文件

**预期**：两个密码各开各的卷，内容不同。

**原理**（排查用）：我们解锁时先试主头（文件偏移 0），开不了再试隐藏卷头（偏移 64KB）。同一个密码框，输哪个密码开哪个卷。

**失败排查**：若隐藏卷密码开不了 → 记下隐藏卷是用什么算法/PRF 建的，告诉我。

---

### 【A2】卷头备份 / 救砖　★写入类（危险，先拿废容器练）
**做什么**：备份卷头到文件；主头损坏后从卷尾备份头救回。

**入口**：解锁区底部"卷头工具（备份/救砖）"。

**测试 1（备份，安全）**：
1. 选中容器（给读权限即可）
2. 卷头工具 → 导出卷头备份 → 存个文件
3. 预期：得到一个 128KB 的备份文件

**测试 2（救砖，危险）**：
1. **先拿一个不要的废容器**（或先在电脑上完整复制一份原容器备着）
2. 在电脑上用十六进制编辑器把容器**开头几百字节**改乱（模拟主头损坏），确认桌面 VC 已打不开
3. 手机选中这个坏容器（**要写权限**）→ 输正确密码 → 卷头工具 → 从内嵌备份头救砖
4. 它会先让你选一个"救援文件"保存位置（这是覆盖前的兜底，万一救砖更糟能退回）
5. 预期：救砖后，桌面 VC 又能用原密码打开了

**原理**：救砖前会先验证"卷尾备份头能用你的密码打开"，开不了就拒绝写（不会拿垃圾覆盖）。

**失败排查**：若救砖后桌面仍打不开 → 把"救援文件"留着（那是你原来的坏主头），告诉我。

---

### 【B1】改密码 / PIM / PRF / keyfile　★写入类（危险）
**做什么**：改容器密码（主密钥不变，只换头，数据不动）。

**入口**：解锁区"修改密码 / PIM / keyfile"（**需写权限**）。

**手机步骤**：
1. **先拿废容器或备份好原容器**
2. 选中容器（要写权限）→ 在**上方**输**旧**密码/PIM/PRF/keyfile（和解锁时一样填法）
3. 点"修改密码"入口 → 弹窗里填**新**密码/PIM/PRF/keyfile
4. 确认时它会先让你选"救援文件"位置（兜底），然后执行
5. 执行后它会自己用新密码试开验证一遍

**预期**：
- 桌面 VC 用**新**密码能开
- 用**旧**密码打不开
- 里面的文件**完全没变**（主密钥没动，数据原样）

**原理**：改密码 = 用旧密码开卷拿到主密钥 → 用新密码重新派生头密钥 → 重加密主头+备份头两处。数据区一个字节都不碰。

**失败排查**：若改完两个密码都打不开 → 用救援文件走 A2 救砖流程恢复，然后告诉我旧/新 PRF 分别选的什么。

---

### 【B2】创建新容器　★写入类
**做什么**：手机上从零建一个 VeraCrypt 容器。

**入口**：主页未选容器时的"创建新容器"。

**手机步骤**：
1. 创建新容器 → 表单页填：算法 / PRF / 大小（先填 10-20MB）/ 密码 / PIM
2. 点"下一步"→ 进**全屏熵页**：手指在屏幕上涂抹（能晃手机更好），看进度条到满
3. 满了点"创建"→ 选文件保存位置

**预期**：
- 桌面 VC 用同密码能挂载
- 挂载后 chkdsk（Windows：`chkdsk X:`）**干净无错**
- 能在里面读写文件

**熵页要点**（见第五章，重要）：
- hex 网格的跳动是**真实熵池**，不是假动画
- **手停下池就不跳了**——这是真实的，不是卡死（我们没有桌面版那种"后台持续采系统熵"，静止时池本就不变）

**失败排查**：若桌面挂载失败 → 记下算法+PRF+大小；若挂载成功但 chkdsk 报错 → 记下报什么错。

---

### 【C2】创建隐藏卷　★写入类（最复杂，重点测）
**做什么**：在已有的外层容器尾部空间里，造一个隐藏卷。

**入口**：解锁区"创建隐藏卷"（**需写权限**）。

**准备**：先有一个外层容器（可用 B2 建的，或桌面 VC 建的普通容器）。外层最好留足空闲空间（隐藏卷要占尾部）。

**手机步骤**：
1. 选中外层容器（要写权限）→ 在**上方**输**外层**密码（解锁外层用，我们要读外层 FAT 算安全区）
2. 点"创建隐藏卷"→ 弹窗填：隐藏卷算法/PRF/大小/**隐藏卷新密码**/PIM
   - 隐藏卷大小要 < 外层空闲空间
3. 确认

**预期**：
- 桌面 VC 用**隐藏卷密码**能开 → 看到空的隐藏卷
- 用**外层密码**仍能开 → 外层原有文件**完好无损**
- 两个卷 chkdsk 都干净

**外层写保护**（重点验证）：
- 这是关键安全机制：创建隐藏卷前，我们读外层 FAT 算出"外层已用到哪"，隐藏卷只放在那之后，**不覆盖外层已有文件**
- **测试方法**：外层先塞几个文件 → 建隐藏卷 → 回头用外层密码开，确认那几个文件还在、能正常读
- 若隐藏卷要的空间和外层已用区冲突，应该**报错拒绝创建**（而不是覆盖外层文件）

**失败排查**（这块最复杂，出问题详细记）：
- 桌面用隐藏卷密码开不了 → 记隐藏卷的算法/PRF/大小、外层容器多大
- 外层文件被破坏 → **严重**，立刻记下外层塞了多大的文件、隐藏卷设多大，这是写保护算错了
- 报错拒绝创建但你觉得空间够 → 记外层总大小/已用/隐藏卷要多大

## 三、四种文件系统容器互通测试（D1/D2/D3/D4）

**这是 v0.6 新增重点**：验证 app 能读写桌面 VC 创建的四种文件系统容器，写回后桌面 VC 仍能打开 + chkdsk 干净。
**红线**：写错一个扇区 = 整个容器报废。每个写入测试前**先在电脑上完整复制一份容器备着**。

### 【D1】FAT16 / FAT32 容器互通　★写入类
**做什么**：桌面 VC 造 FAT 卷 → app 挂载读写 → 回桌面 VC + chkdsk 验。

**准备（桌面 VC 造两个卷）**：
1. 桌面 VC 创建容器 A：算法 AES(1) / PRF SHA-512(1) / 大小 10MB / 文件系统选 **FAT**（VC 自动按大小选 FAT16）→ 格式化后挂载，塞一个 `hello.txt`
2. 桌面 VC 创建容器 B：算法 AES(1) / PRF SHA-512(1) / 大小 50MB / 文件系统选 **FAT**（VC 自动选 FAT32）→ 挂载，塞一个 `world.txt`

**手机步骤（每个卷都做一遍）**：
1. 选中容器（**要写权限**）→ 输密码 → 解锁 → 应看到塞进去的文件
2. 导出该文件到手机 → 对比内容一致
3. 用 app **导入**一个新文件 `phone.txt`（几十 KB 即可）
4. 用 app **删除** `hello.txt`（或 `world.txt`）
5. 用 app **覆写**：用手机文件覆写剩余的那个文件
6. 上锁 → 把容器传回电脑

**电脑回验**：
1. 桌面 VC 用原密码挂载 → 应看到 `phone.txt` + 被覆写的那个文件，原删除的文件不在
2. 卸载后 Windows 跑 `chkdsk X:` → 应**干净无错**（FAT32 可能报"空闲空间不符已修正"非致命警告，记下）
3. 打开 `phone.txt` + 被覆写文件 → 内容正确

**失败排查**：
- app 解锁失败 → 记容器是 FAT16 还是 FAT32（看 BPB），算法/PRF
- 写入后桌面 VC 打不开 → **严重**，立刻停止，把容器和"救援文件"留着
- chkdsk 报错 → 记具体输出

### 【D2】exFAT 容器互通　★写入类
**做什么**：桌面 VC 造 exFAT 卷 → app 挂载读写 → 回桌面 VC + chkdsk 验。

**准备（桌面 VC 造一个 exFAT 卷）**：
- 桌面 VC 创建容器：算法 AES(1) / PRF SHA-512(1) / 大小 50MB / 文件系统选 **exFAT** → 格式化后挂载，塞一个 `exfat.txt`

**手机步骤**：
1. 选中容器（**要写权限**）→ 输密码 → 解锁 → 应看到 `exfat.txt`，文件系统显示"exFAT"
2. 导出 `exfat.txt` → 对比内容
3. app 导入新文件 `phone_exfat.txt`
4. app 删除 `exfat.txt`
5. **重点测（X4 审计发现的 bug 场景）**：app 删除桌面 VC 创建的**大文件**（连续分配 / NoFatChain）
   - 桌面 VC 先在卷里塞一个**占满大半卷**的大文件（如 30MB 卷塞 20MB 文件）→ 这种大文件桌面 VC 会用 NoFatChain 连续分配
   - app 解锁 → 删除该大文件 → 上锁 → 回桌面 VC
   - **预期**：桌面 VC 挂载后该文件已删 + chkdsk 干净 + 空闲空间恢复正确（若空闲空间没恢复 = X4 BUG-1 未修，簇泄漏）
   - **若 M 尚未修 X4 BUG-1**：此步会失败（空闲空间不恢复），记下后跳过，等 M 修完再回测
6. 上锁 → 容器传回电脑

**电脑回验**：
1. 桌面 VC 挂载 → 文件状态符合预期
2. `chkdsk X:` → 干净无错（exFAT 的 chkdsk 会校验分配位图 + FAT 表一致性）
3. 打开文件 → 内容正确

**失败排查**：
- exFAT 卷解锁失败 → 记 boot 扇区字段（簇大小、根簇号），告诉我
- 删除大文件后空闲空间没恢复 → X4 BUG-1 簇泄漏，确认 M 是否已修
- chkdsk 报"分配位图损坏" → exFAT 位图写错，记报错

## 四、自动锁定测试（E 阶段）

**v0.6 新增**：验证容器解锁后，超时/切后台/息屏能自动上锁（销毁内存密钥）。

**入口**：主屏右上角齿轮图标 → 设置弹窗。

### 【E1】超时自动锁定
**做什么**：解锁容器后，等超时到期，确认自动上锁。

**手机步骤**：
1. 解锁一个容器（进入挂载态）
2. 进设置 → 自动锁定开关确认**开** → 超时档位选**1 分钟**
3. 退出设置，**不动手机**，等 1 分钟
4. 预期：1 分钟后自动回到锁定态（容器关闭，内存密钥销毁）

**关键对照**：
- 超时前任意触摸/滑动 → 计时重置（onUserInteraction → AutoLock.touch）
- 超时档位"关闭" → 不触发超时锁（仅靠切后台/息屏）

**失败排查**：
- 超时不触发 → 记超时档位、是否触摸过；可能 AutoLock 计时器没起
- 触摸不重置 → onUserInteraction override 没生效

---

### 【E2】切后台自动锁定
**做什么**：解锁后切到后台（按 Home / 切别的 app），确认立即上锁。

**手机步骤**：
1. 解锁容器
2. 进设置 → "切到后台时立即锁定"开关确认**开**
3. 按 Home 键 / 切到别的 app
4. 切回匿匣
5. 预期：切回时已是锁定态（容器关闭）

**关键对照**：
- 开关关 → 切后台不锁（仍靠超时）
- 此项依赖 lifecycle-process 监听 ON_STOP 事件

**失败排查**：
- 切后台不锁 → 记设置开关状态；可能 SealchestApp 的 LifecycleObserver 没接上

---

### 【E3】息屏自动锁定
**做什么**：解锁后按电源键息屏，确认上锁。

**手机步骤**：
1. 解锁容器
2. 进设置 → "息屏时立即锁定"开关确认**开**
3. 按电源键息屏 → 等几秒 → 亮屏
4. 预期：亮屏后已是锁定态

**关键对照**：
- 开关关 → 息屏不锁
- 默认此开关**关**（超时 + 切后台默认开，息屏默认关——避免频繁锁）

**失败排查**：
- 息屏不锁 → 记开关状态；可能 ACTION_SCREEN_OFF 广播没注册

---

### 【E4】设置持久化
**做什么**：改设置 → 杀进程 → 重开 → 确认设置保留。

**手机步骤**：
1. 解锁容器 → 进设置 → 改几个开关 + 超时档位
2. 退出设置 → 上锁 → 杀掉匿匣进程（系统设置 → 应用 → 强行停止，或从最近任务划掉）
3. 重新打开匿匣 → 进设置
4. 预期：刚才改的设置都在（SharedPreferences 持久化）

**失败排查**：
- 设置丢失 → SharedPreferences 没写或 key 不对

## 五、熵池跳动是真的吗？

**是真的。** 数据链路（我核过真实代码）：
- 你每涂抹一下 / 晃一下手机 → 触点坐标+纳秒时间戳 / 传感器读数，喂进 native 的 320 字节熵池
- 喂的时候：每个字节经 CRC32 扩散后模加进池，每 16 字节触发一次 SHA-512 搅拌整池
- 屏幕上的 hex 每 80ms 从**同一个熵池**原样读出来显示
- 这个池就是接下来生成容器主密钥的那个池

**和桌面 VC 的真实差异（诚实说明）**：
- 桌面 VC 你不动鼠标，池也在跳——因为它后台持续采集系统熵（FastPoll）
- 我们的池**只在你涂抹/晃手机时才跳，手停就静止**
- 原因：我复刻了 VC 的熵池搅拌算法，但没复刻"持续采系统熵"那一路（那是 Windows 特定的熵源采集）
- 我没有用假动画糊弄——静止就是静止，跳就是真在变

**这点要不要改**：想让静止时也跳，只能持续灌系统时钟之类，但那注入的不是你的物理熵、有点自欺。要不要加、怎么加，等你测完一起定。

## 八、NTFS 实验功能验收指南

> **背景**：NTFS 读写是新完成的高危功能（B+树整树重建 + 造盘器）。当前 MountManager 默认**拒绝挂载 NTFS**，须用户在设置里开启「NTFS 实验」开关才能用。
> **唯一判据**：恒烈真机验收。读通过才许写。

### 8.1 验收纪律（必须遵守）

- **先只读后可写**：
  1. 开 NTFS 实验开关 → 选桌面 VC 造的 NTFS 容器**只读挂载** → 验证能挂 / 浏览 / 导出
  2. 只读全过，才允许**可写挂载**测新建 / 写 / 删 / 新建文件夹 / 大目录分裂
- **全程用废容器**：丢弃数据也不心疼的测试容器；写入前先在电脑上完整复制一份备着
- **NTFS 实验开关位置**：设置 → 「启用 NTFS（实验）」，默认关

### 8.2 验收步骤

#### 阶段一：只读验收

1. 桌面 VeraCrypt 建 NTFS 容器，记录：算法 / PRF / 大小 / 卷标
2. 推到手机，app 里开 NTFS 实验开关
3. **只读挂载**（不勾「可写挂载」）
4. 验证：
   - 能挂载
   - 能浏览目录
   - 能导出文件
   - 导出文件与桌面 VC 内容**字节级对比一致**

#### 阶段二：可写验收（只读全过后才做）

1. 可写挂载同一容器
2. 逐项测试（**每项单独测，别攒一起**）：
   - 新建文件 → 上锁 → 桌面 VC 回验
   - 写入/覆盖文件 → 上锁 → 桌面 VC 回验
   - 删除文件 → 上锁 → 桌面 VC 回验
   - 新建文件夹 → 上锁 → 桌面 VC 回验
3. 大目录测试（**>50 文件触发 B+树两层分裂**）：
   - 在 app 里连续新建 60+ 文件
   - 上锁 → 桌面 VC 挂载 + 跑 `chkdsk X:`（X = 桌面 VC 挂载盘符）
4. **每步回验**：上锁后用桌面 VeraCrypt 打开同容器，确认改动可见 + 文件系统无损

### 8.3 回执要求（验收时提交）

- ① 桌面 VC 建 NTFS 容器的算法 / PRF / 大小
- ② app 能否只读挂载 + 浏览 + 导出对比一致
- ③ 可写测试各操作后桌面 VC 能否打开 + `chkdsk X:` 完整输出
- ④ 大目录（>50 文件触发 2 层分裂）建后 `chkdsk X:` 输出
- ⑤ 出错把「救援文件」留着 + 容器**别删**

### 8.4 出错报告格式

出错时按下列项记全，发我：

- 哪步出错（阶段一/阶段二、具体操作）
- 容器参数：算法 / PRF / 大小 / 卷标
- 桌面 VC 报错原文
- `chkdsk X:` 原文
- 是读路径还是写路径
