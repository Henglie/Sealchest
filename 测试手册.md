# 密匣 Sealchest · 测试手册（v0.5 待验收）

> 本文档给恒烈真机验收用。git HEAD `3a9f0c4`。所有功能已编译过、未真机验。
> **唯一判据**：与桌面 VeraCrypt 字节级互通。凡涉及"桌面 VC 能否打开/chkdsk 是否干净"的，以桌面为准。

## 一、环境准备

### 1.1 你需要
- 一加手机（安卓 16），装本轮 APK：`Sealchest-C2隐藏卷创建.apk`（在桌面）
- 电脑上的桌面版 VeraCrypt（1.26.x，与我们上游 pin 的 1.26.29 同代）
- 一个能在手机/电脑间传文件的方式（数据线 / 网盘 / adb push）
- **测试用密码，别用你真实密码**；测试容器建小的（10-50MB）

### 1.2 手机上给容器授权（重要）
- 选容器时，系统会问读权限——必须给
- 想测**写入类功能**（改密码 / 创建隐藏卷 / 救砖），选容器时还得让它拿到**写权限**：
  这些入口只有在"选中的容器 URI 拿到了持久写权限"时才显示。拿不到写权限，这些按钮不出现（不是 bug）。
- 若某入口没出现，先确认：选的是可写位置的容器（如手机内部存储 / SD 卡），不是只读来源（如某些云盘挂载）

### 1.3 算法 / PRF 编号对照（排查时用）
创建容器时选的算法/PRF，对应 VeraCrypt 编号：

| 算法 | 编号 | | PRF | 编号 |
|---|---|---|---|---|
| AES | 1 | | SHA-512 | 1 |
| Serpent | 2 | | Whirlpool | 2 |
| Twofish | 3 | | SHA-256 | 3 |
| Camellia | 4 | | BLAKE2s | 4 |
| Kuznyechik | 5 | | Streebog | 5 |

解锁时可选"自动"PRF（挨个试）；创建时必须指定确定 PRF。

## 二、待验功能清单（建议按此顺序测）

优先级：先测**只读/解锁类**（不改容器，安全），再测**写入类**（会改容器，先拿废容器练手）。

### 【A1】keyfile 解锁　★只读类
**做什么**：用密码 + 一个或多个 keyfile 解锁容器。

**准备**：桌面 VC 造一个带 keyfile 的容器（创建时勾 Keyfiles，选定几个文件当钥匙）。记住密码 + 那几个 keyfile。

**手机步骤**：
1. 选中该容器 → 输密码
2. 点"密钥文件（可选）"→ 选中**同一组** keyfile（顺序无所谓）
3. 解锁

**预期**：能开，能浏览里面的文件。

**关键对照**：
- 同一组 keyfile 在手机和桌面结果必须一致（都能开或都不能开）
- 少选/多选一个 keyfile → 应该打不开（keyfile 是全集匹配）
- 纯 keyfile 无密码：VC 允许空密码 + keyfile，我们也放开了空密码解锁

**失败排查**：若桌面能开、手机不能开 → 可能是 keyfile 混入算法（CRC32 池化）有偏差，记下用了几个 keyfile、文件多大，告诉我。

---

### 【C1】隐藏卷解锁　★只读类
**做什么**：同一个容器，输外层密码开外层卷，输隐藏卷密码开隐藏卷。

**准备**：桌面 VC 造一个带隐藏卷的容器（Create → Hidden Volume）。外层放几个诱饵文件，隐藏卷放几个不同的文件。记两个密码。

**手机步骤**：
1. 选中容器 → 输**外层**密码 → 解锁 → 应看到诱饵文件
2. 上锁 → 重新选中 → 输**隐藏卷**密码 → 解锁 → 应看到隐藏卷的文件

**预期**：两个密码各开各的卷，内容不同。

**原理**（排查用）：我们解锁时先试主头（文件偏移 0），开不了再试隐藏卷头（偏移 64KB）。同一个密码框，输哪个密码开哪个卷。

**失败排查**：若隐藏卷密码开不了 → 记下隐藏卷是用什么算法/PRF 建的，告诉我。

---

### 【A2】卷头备份 / 救砖　★写入类（危险，先拿废容器练）
**做什么**：备份卷头到文件；主头损坏后从卷尾备份头救回。

**入口**：解锁区底部"卷头工具（备份/救砖）"。

**测试 1（备份，安全）**：
1. 选中容器（给读权限即可）
2. 卷头工具 → 导出卷头备份 → 存个文件
3. 预期：得到一个 128KB 的备份文件

**测试 2（救砖，危险）**：
1. **先拿一个不要的废容器**（或先在电脑上完整复制一份原容器备着）
2. 在电脑上用十六进制编辑器把容器**开头几百字节**改乱（模拟主头损坏），确认桌面 VC 已打不开
3. 手机选中这个坏容器（**要写权限**）→ 输正确密码 → 卷头工具 → 从内嵌备份头救砖
4. 它会先让你选一个"救援文件"保存位置（这是覆盖前的兜底，万一救砖更糟能退回）
5. 预期：救砖后，桌面 VC 又能用原密码打开了

**原理**：救砖前会先验证"卷尾备份头能用你的密码打开"，开不了就拒绝写（不会拿垃圾覆盖）。

**失败排查**：若救砖后桌面仍打不开 → 把"救援文件"留着（那是你原来的坏主头），告诉我。

---

### 【B1】改密码 / PIM / PRF / keyfile　★写入类（危险）
**做什么**：改容器密码（主密钥不变，只换头，数据不动）。

**入口**：解锁区"修改密码 / PIM / keyfile"（**需写权限**）。

**手机步骤**：
1. **先拿废容器或备份好原容器**
2. 选中容器（要写权限）→ 在**上方**输**旧**密码/PIM/PRF/keyfile（和解锁时一样填法）
3. 点"修改密码"入口 → 弹窗里填**新**密码/PIM/PRF/keyfile
4. 确认时它会先让你选"救援文件"位置（兜底），然后执行
5. 执行后它会自己用新密码试开验证一遍

**预期**：
- 桌面 VC 用**新**密码能开
- 用**旧**密码打不开
- 里面的文件**完全没变**（主密钥没动，数据原样）

**原理**：改密码 = 用旧密码开卷拿到主密钥 → 用新密码重新派生头密钥 → 重加密主头+备份头两处。数据区一个字节都不碰。

**失败排查**：若改完两个密码都打不开 → 用救援文件走 A2 救砖流程恢复，然后告诉我旧/新 PRF 分别选的什么。

---

### 【B2】创建新容器　★写入类
**做什么**：手机上从零建一个 VeraCrypt 容器。

**入口**：主页未选容器时的"创建新容器"。

**手机步骤**：
1. 创建新容器 → 表单页填：算法 / PRF / 大小（先填 10-20MB）/ 密码 / PIM
2. 点"下一步"→ 进**全屏熵页**：手指在屏幕上涂抹（能晃手机更好），看进度条到满
3. 满了点"创建"→ 选文件保存位置

**预期**：
- 桌面 VC 用同密码能挂载
- 挂载后 chkdsk（Windows：`chkdsk X:`）**干净无错**
- 能在里面读写文件

**熵页要点**（见第三节，重要）：
- hex 网格的跳动是**真实熵池**，不是假动画
- **手停下池就不跳了**——这是真实的，不是卡死（我们没有桌面版那种"后台持续采系统熵"，静止时池本就不变）

**失败排查**：若桌面挂载失败 → 记下算法+PRF+大小；若挂载成功但 chkdsk 报错 → 记下报什么错。

### 【C2】创建隐藏卷　★写入类（最复杂，重点测）
**做什么**：在已有的外层容器尾部空间里，造一个隐藏卷。

**入口**：解锁区"创建隐藏卷"（**需写权限**）。

**准备**：先有一个外层容器（可用 B2 建的，或桌面 VC 建的普通容器）。外层最好留足空闲空间（隐藏卷要占尾部）。

**手机步骤**：
1. 选中外层容器（要写权限）→ 在**上方**输**外层**密码（解锁外层用，我们要读外层 FAT 算安全区）
2. 点"创建隐藏卷"→ 弹窗填：隐藏卷算法/PRF/大小/**隐藏卷新密码**/PIM
   - 隐藏卷大小要 < 外层空闲空间
3. 确认

**预期**：
- 桌面 VC 用**隐藏卷密码**能开 → 看到空的隐藏卷
- 用**外层密码**仍能开 → 外层原有文件**完好无损**
- 两个卷 chkdsk 都干净

**外层写保护**（重点验证）：
- 这是关键安全机制：创建隐藏卷前，我们读外层 FAT 算出"外层已用到哪"，隐藏卷只放在那之后，**不覆盖外层已有文件**
- **测试方法**：外层先塞几个文件 → 建隐藏卷 → 回头用外层密码开，确认那几个文件还在、能正常读
- 若隐藏卷要的空间和外层已用区冲突，应该**报错拒绝创建**（而不是覆盖外层文件）

**失败排查**（这块最复杂，出问题详细记）：
- 桌面用隐藏卷密码开不了 → 记隐藏卷的算法/PRF/大小、外层容器多大
- 外层文件被破坏 → **严重**，立刻记下外层塞了多大的文件、隐藏卷设多大，这是写保护算错了
- 报错拒绝创建但你觉得空间够 → 记外层总大小/已用/隐藏卷要多大

## 三、熵池跳动是真的吗？（你问的）

**是真的。** 数据链路（我核过真实代码）：
- 你每涂抹一下 / 晃一下手机 → 触点坐标+纳秒时间戳 / 传感器读数，喂进 native 的 320 字节熵池
- 喂的时候：每个字节经 CRC32 扩散后模加进池，每 16 字节触发一次 SHA-512 搅拌整池
- 屏幕上的 hex 每 80ms 从**同一个熵池**原样读出来显示
- 这个池就是接下来生成容器主密钥的那个池

**和桌面 VC 的真实差异（诚实说明）**：
- 桌面 VC 你不动鼠标，池也在跳——因为它后台持续采集系统熵（FastPoll）
- 我们的池**只在你涂抹/晃手机时才跳，手停就静止**
- 原因：我复刻了 VC 的熵池搅拌算法，但没复刻"持续采系统熵"那一路（那是 Windows 特定的熵源采集）
- 我没有用假动画糊弄——静止就是静止，跳就是真在变

**这点要不要改**：想让静止时也跳，只能持续灌系统时钟之类，但那注入的不是你的物理熵、有点自欺。要不要加、怎么加，等你测完一起定。
